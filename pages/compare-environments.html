Compare Environments

The Compare Environments tool helps you identify differences between two environments by comparing file checksums in a directory.

How to Use

  1. Download the latest version of the tool by clicking the "Download Latest Version" button above.
  2. Run the checksum application on the first directory you want to compare.
    • Select the directory to checksum.
    • Choose the number of workers (threads).
    • Choose the algorithm (ensure this is the same for both directories).
  3. Run the checksum application on the second directory with the same settings.
  4. Run the compare application to process the outputs from the checksum steps.
  5. Analyze the results:
    • The output will be a CSV file showing the differences, including missing or modified files.
    • The file structure will be preserved for easy navigation.

Example Workflow

      1. Run the checksum application on directory A:
         - Directory: /path/to/directoryA
         - Algorithm: SHA256
         - Workers: 4

      2. Run the checksum application on directory B:
         - Directory: /path/to/directoryB
         - Algorithm: SHA256
         - Workers: 4

      3. Run the compare application:
         - Input: DirectoryA and DirectoryB

In this example, the Compare Environments tool will output 2 CSV files, differences.csv and missing.csv, showing the missing and modified files in the two directories.

Best Practices

  • Always use the same algorithm (e.g., SHA256) for both directories to ensure accurate comparisons.
  • Use an appropriate number of workers based on your system's performance.
  • Organize the output files in a dedicated folder for easier processing.
