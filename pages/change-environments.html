How to Use

  1. Download the latest version of the tool by clicking the "Download Latest Version" button above.
  2. Ensure you have the required customer configuration folder. If you don't have it, you can download the default configuration by clicking the button below:
  3. Place the configuration folder inside the same directory as the tool executable.
  4. Run the tool and follow the prompts or graphical interface to change or compare environments.
  5. If you want to create your own configuration, follow the example directory tree and YAML format provided below.

Example Directory Tree

      customers/
      ├── QAS/
      │   ├── Project23/
      │   │   ├── interface.ps1
      │   │   ├── workstation.ps1
      │   ├── QAS.yaml
      └── customers.yaml

In the directory above:

  • customers.yaml: Contains a list of all customers.
  • QAS.yaml: Defines environments (e.g., "Project23") and associated scripts.
  • Each environment folder (e.g., "Project23") contains the specific scripts (e.g., "interface.ps1").

YAML File Example

      QAS:
        - Project23:
          - Version: "23.1.3.3"
          - Workstation: "workstation.ps1"
          - Interface: "interface.ps1"

This YAML defines the customer "QAS" and its environments. Each environment has associated scripts for specific operations.
